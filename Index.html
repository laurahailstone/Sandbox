HTML Notes

HTML (Hypertext Markup Language) a standardized system for tagging text files to achieve font, color, graphic, and hyperlink effects on World Wide Web pages.

HTML Elements

An HTML element is an individual component of an HTML document It's function is to represent the meaning/semantics of the content. Most HTML elements start and end with "tags" <> and . These tags surround and give meaning to the content. For example, this element is a paragraph. It is surrounded by p tags.

Some tags give a descriptive structure to your HTML which makes it easier to read and help with Search Engine Optimization. Some examples include:

  • Headers
  • Paragraph
  • Image
  • Link
  • Unordered List (like this one!)
  • Ordered Lists
  • List Items

  • Line Break (did you catch that?)
...and many more! There are so many elements! Thankfully, elements can be nested within other elements. For example, the "unordered list" and "list item" elements have been nested within this paragraph!

There is even a tag which allows you to make comments without them displaying with the rest of your content. Look.... See? No, you can't see it. ...but now you can.

Structural Elements

Some HTML elements are used purely for structural purposes.

  • Div Element - This element defines a section and contains other elements which can then be stylized by CSS or manipulated by JavaScript.

  • !DOCTYPE... Element - This element tells the browser which version of HTML is being used. Remember that the tag must be capitalized.

  • html Element- This element wraps around all other html elements and acts as a container for the whole document.


HTML5 Elements

HTML5 introduces more description HTML tags which makes it easier to read and help with Search Engine Optimization. Some examples include:

  • main
  • header
  • footer
  • nav
  • video
  • article
Now it can be confusing to know the difference among these structural elements, so here is some help.

Head tag=information ABOUT your page
Body tag =all the contents of an HTML document, such as text, hyperlinks, images, tables, lists, etc.
Main tag=main content of the document. Should be unique to the document and not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
Footer tag=usually copyright, authorship, contact information, site map, and links back to the top.

HTML Attributes

We can provide more information about HTML elements by assigning them attributes. All HTML elements can have attributes, and the attributes are always specified in the opening tag. For example, the h4 header of this secion has been assigned the style attribute of "color:purple;. Here are some other attributes that can be specified:

  • href= This defines the hyperlink.
  • src= This is used to embed an image into the code. It specifis the path to the image to be displayed.
  • height= and width=Used to define the dimensions (in pixels) of an image element.
  • alt= This defines defines the alternate text to an image element.
  • style= This defines the color, font, size, and more.
  • id= This specifies a unique ID for an html element.


Images and Links

You can add images to your website by using the img element, and point to a specific image's URL using the src attribute. Remember that the image element is self-closing, which means you don't need /img inside the carrots. Also, every image requires an alt attribute which populates the space with text in the event that the image does not load. Even if the image is only for decorative purposes, the alt attribute is required but can be left blank. (ex. alt="")

You can insert a link using text or an image. Here is an example of linking via text. Value Attribute and Data Collection

When a form is submitted, the data is sent to a server and includes the entries for the options selected. For radio and checkbox input types, the data that is reported comes from the value attribute.

Let's suppose that a user completes the form above, and selects "Radio Button" as their favorite input type. Because there is a value attribute assigned to the input element, the data "Radio Button" will be reported. If the value attribute is missing, then a default value of "on" will be reported, but it is not useful information.

If you have a webform with radio buttons or checkboxes, ALWAYS REMEMBER TO INCLUDE VALUE ATTRIBUTES!
